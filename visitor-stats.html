How Visitor Tracking Works

1

Total Views — Incremented only when someone visits the Home page. Each browser session counts once (no duplicate counting on refresh).

2

Online Users — Uses Firebase presence system. When you open the site, you're marked as "online". When you close the tab, you're automatically removed.

3

Real-time — All numbers update instantly across all connected browsers without any page refresh needed.

4

Privacy — No personal data is collected. Only anonymous session IDs are used for counting purposes.

Firebase Database Structure 

{
  "visitorTracking": {
    "totalViews": 1234,
    "presence": {
      "sess_17089...a3f": true,
      "sess_17089...b7e": true
    }
  }
}

Recommended Database Rules 

{
  "rules": {
    "visitorTracking": {
      "totalViews": {
        ".read": true,
        ".write": true,
        ".validate": "newData.isNumber()"
      },
      "presence": {
        "$sessionId": {
          ".read": true,
          ".write": true,
          ".validate": "newData.val() === true"
        }
      }
    }
  }
}

These rules allow anyone to read/write visitor tracking data (required for anonymous tracking on a static site). The validation ensures only valid data types are stored.
